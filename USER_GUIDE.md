## Profile Management

### Updating Profile Picture

1. **Navigate to Profile Tab**
2. **Click "Choose File" under Profile Picture**
3. **Select image from your computer**
   - Supported formats: PNG, JPG, GIF
   - Recommended size: 800x800px minimum
   - Square images work best
4. **Preview appears immediately**
5. **Click "Save All Changes"**

**Tips:**
- Use a professional headshot
- Ensure good lighting and clear background
- Square aspect ratio prevents cropping issues
- File size under 2MB loads faster

### Editing Basic Information

**Fields Available:**
- **Full Name** - Your complete name
- **Job Title** - Current position/role
- **Tagline** - One-line professional description
- **Professional Summary** - 2-3 paragraph overview
- **About Me** - Personal background and interests

**How to Edit:**
1. Click in any text field
2. Type your changes
3. Use Enter for new paragraphs in text areas
4. Click "Save All Changes" when done

### Managing Contact Information

**Available Fields:**
- Email address
- Phone number
- Location (city, country)
- LinkedIn URL (full link)
- GitHub URL (full link)
- Website URL (optional)

**Best Practices:**
- Use professional email address
- Include country code in phone (e.g., +60-XXX-XXX-XXXX)
- Full LinkedIn URL: `https://www.linkedin.com/in/yourprofile`
- Full GitHub URL: `https://github.com/yourusername`

### Adding Statistics

Statistics appear as badges on your homepage.

**Examples:**
- `6+` Years Experience
- `12+` Markets Served
- `100+` Projects Completed
- `50+` Technologies Used

**To Add:**
1. Click "Add Stat" button
2. **Value** field: Enter number with + (e.g., "6+")
3. **Label** field: Enter description (e.g., "Years Experience")
4. Repeat for multiple stats
5. Click "Save All Changes"

**To Edit/Delete:**
- Click in fields to edit
- Click trash icon to delete
- Save changes

### Managing Key Highlights

Highlights appear as bullet points on homepage.

**Examples:**
- "Led migration of enterprise applications to .NET 8"
- "Developed serverless applications serving millions of requests"
- "Implemented AI-driven features including sentiment analysis"

**To Add:**
1. Click "Add Highlight" button
2. Type your highlight in text field
3. Keep it concise (1-2 lines)
4. Repeat for multiple highlights
5. Click "Save All Changes"

**To Delete:**
- Click trash icon next to highlight
- Save changes

## Experience Management

### Adding New Experience

1. **Click "Add Experience" button**
2. **Fill in all fields:**
   - **Job Title** - Your position (e.g., "Senior Full Stack Developer")
   - **Company** - Company name
   - **Location** - City, Country
   - **Start Date** - Format: "Month Year" (e.g., "January 2024")
   - **End Date** - "Present" or "Month Year"
   - **Company Type** - Brief description (e.g., "Technology Company")
   - **Description** - Company overview (2-3 lines)
   - **Achievements** - One per line, major accomplishments
   - **Technologies** - Comma-separated list

3. **Click "Save All Changes"**

### Editing Existing Experience

1. **Find the experience entry**
2. **Click in any field to edit directly**
3. **Achievements:** Each line = one achievement
4. **Technologies:** Separate with commas, no spaces
5. **Click "Save All Changes"**

### Deleting Experience

1. **Click "Delete" button on experience card**
2. **Confirm deletion**
3. **Click "Save All Changes"**

**Note:** Deleted entries cannot be recovered. Copy text before deleting if you might need it later.

### Experience Best Practices

**Dates:**
- Use consistent format: "January 2024"
- For current role: Use "Present"
- Month first, then year

**Achievements:**
- Start with action verbs (Led, Developed, Implemented)
- Include metrics when possible (e.g., "serving millions of users")
- Focus on impact and results
- 3-5 achievements per role

**Technologies:**
- List most relevant first
- Use official names (.NET 8, not "dot net eight")
- Comma-separated: `.NET 8, React, AWS Lambda`
- 5-10 technologies per role

## Project Management

### Adding New Project

1. **Click "Add Project" button**
2. **Fill in fields:**
   - **Project Name** - Official project name
   - **URL** - Live project link (https://...)
   - **Status** - Current state (Active, In Development, etc.)
   - **Market** - Target region (e.g., "Bangladesh & Malaysia")
   - **Description** - What the project does (2-3 sentences)
   - **Features** - One feature per line
   - **Technologies** - Comma-separated
   - **Image URL** - Project screenshot URL (optional)

3. **Click "Save All Changes"**

### Editing Projects

1. **Find project in list**
2. **Click in fields to edit**
3. **Features:** One per line
4. **Technologies:** Comma-separated
5. **Save changes**

### Deleting Projects

1. **Click "Delete" on project card**
2. **Confirm**
3. **Save changes**

### Project Best Practices

**URLs:**
- Always include `https://`
- Test link before saving
- Use live production URLs when possible

**Descriptions:**
- Clear, concise explanation
- Focus on value proposition
- Mention key functionality

**Features:**
- 3-6 key features
- User-facing benefits
- Specific functionality
- One per line for clarity

**Images:**
- Use high-quality screenshots
- Show actual project interface
- Upload to image hosting service first
- Paste image URL in field

## Skills Management

### Understanding Skill Categories

**8 Categories Available:**
1. **Backend** - Server-side technologies
2. **Frontend** - Client-side technologies
3. **Cloud** - Cloud platforms and services
4. **Databases** - Database systems
5. **Testing** - Testing and DevOps tools
6. **Tools** - Development tools
7. **CMS** - Content management systems
8. **AI** - AI and ML technologies

### Adding Skills

1. **Navigate to desired category card**
2. **Click "Add Skill" button**
3. **Fill in three fields:**
   - **Skill Name** - Technology name (e.g., "React")
   - **Proficiency %** - 0-100 (e.g., 90 for expert)
   - **Years** - Years of experience (e.g., 5)

4. **Repeat for more skills**
5. **Click "Save All Changes"**

### Editing Skills

1. **Find skill in category**
2. **Click in field to edit:**
   - Name
   - Proficiency (0-100)
   - Years of experience
3. **Save changes**

### Deleting Skills

1. **Click trash icon next to skill**
2. **Save changes**

### Proficiency Guidelines

**Percentage Levels:**
- **90-100%** - Expert (5+ years, deep expertise)
- **75-89%** - Advanced (3-5 years, production experience)
- **60-74%** - Intermediate (1-3 years, solid understanding)
- **40-59%** - Beginner (< 1 year, basic knowledge)
- **< 40%** - Fundamental (learning, not production-ready)

**Years of Experience:**
- Count only professional/production experience
- Round to nearest whole year
- Include freelance and personal projects if substantial

### Skills Display

Skills appear with:
- Animated progress bar
- Proficiency percentage
- Years of experience badge (if provided)
- Organized by category

## Certifications Management

### Adding Certifications

1. **Click "Add Certification" button**
2. **Fill in fields:**
   - **Certification Name** - Full certificate title
   - **Issuer** - Organization that issued it
   - **Date Issued** - "Month Year"

**Name:**
- Use official certification name
- Include level if applicable (e.g., "AWS Certified Developer - Associate")

**Issuer:**
- Official organization name (e.g., "Amazon Web Services")

**Date:**
- Month and year issued
- Update if renewed

**Credential URL:**
- Link to verify certificate

- Test link before saving

## Best Practices

### General Editing Tips

**Save Often:**
- Click "Save All Changes" after each section
- Browser may lose unsaved data if closed
- No auto-save feature currently

**Consistent Formatting:**
- Use same date format throughout (Month Year)
- Consistent capitalization
- Professional language and tone

**Keep Current:**
- Update monthly with new skills/projects
- Remove outdated information
- Refresh dates for current roles

**Test Changes:**
- Visit public portfolio after saving
- Check on mobile device
- Verify all links work

### Content Quality

**Professional Tone:**
- Third person for bio/summary
- Active voice for achievements
- Specific, measurable results

**Grammar & Spelling:**
- Proofread before saving
- Use spell-check
- Have someone review content

**Completeness:**
- Fill all required fields
- Don't leave "New Position" or placeholder text
- Every entry should add value

### Performance Tips

**Image Sizes:**
- Compress large images before upload
- Under 500KB recommended
- Use online tools like TinyPNG

**Link Verification:**
- Test all URLs before saving
- Check project links regularly
- Update if sites move

**Mobile Preview:**
- View portfolio on phone after updates
- Ensure text readable on small screens
- Check image scaling

## Troubleshooting

### Can't Login

**Solutions:**
- Verify email: `admin@abdullahfaruque.com`
- Check password (case-sensitive): `Admin@123456`
- Clear browser cache and cookies
- Try incognito/private window

### Changes Not Appearing

**Solutions:**
1. Click "Save All Changes" button
2. Wait 2-3 seconds for save confirmation
3. Refresh public portfolio page (Ctrl/Cmd + R)
4. Clear browser cache if needed

### Profile Picture Not Uploading

**Solutions:**
- Check file size (under 5MB)
- Use supported format (PNG, JPG, GIF)
- Try different image
- Compress image before upload

### Lost Session / Logged Out

**Cause:** Session timeout after inactivity

**Solution:**
- Login again
- Continue editing
- Save changes you've made

### Formatting Issues

**Text Areas:**
- Use Enter key for new lines
- Don't use tabs (may cause issues)
- Copy from plain text editor if formatting problems

**Lists:**
- One item per line for achievements/features
- Comma-separated for technologies (no spaces)

### Can't Delete Item

**Possible Issues:**
- Item may be in use elsewhere
- Try saving changes first
- Refresh page and try again

### Contact Support

If issues persist:
1. Note exactly what you were trying to do
2. Take screenshot of error (if any)
3. Check browser console for errors (F12)
4. Document steps to reproduce
